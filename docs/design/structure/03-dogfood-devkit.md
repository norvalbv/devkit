# Piece 3 — Dogfood: devkit governs its own `cli/` + `gate-engine/`

> The acceptance proof: the structure engine that was hardcoded to frink's six Electron `.ts` trees now
> governs devkit's own two `.mjs` trees, declared once in `guard.config.json` and enforced live by
> eslint. Builds on Piece 1 (the walker + config schema, `c7c7c66`) by adding the SECOND derived
> artifact — the eslint rule — so declare-once is complete: one spec → baseline walk + lint rule.

## What shipped

1. **`compileToEslint(treeSpec, exts, {baseline, exempt})`** (`gate-engine/structure/compile.mjs`) —
   translates a grammar tree into an `eslint-plugin-project-structure` `createFolderStructure(...)`
   config: `{token}` → regexParameters, `folders` → child rules, `recurse` → the `rules` map,
   `domainGate` → a closed-registry `{<key>_domain}` alternation, `ignoredDirs`/`__tests__` →
   ignorePatterns, baseline+exempt → ignorePatterns.

2. **No-drift made canonical.** `grammar.mjs` now has ONE `tokenRegex(token, exts)` — the walker
   predicate compiles it (`new RegExp(tokenRegex(...))`, cached) and `compileToEslint` emits the SAME
   string as a regexParameter. Rule and baseline can't disagree about what a `{kebab}` is.
   `structure-compile.test.mjs` pins predicate == emitted regex on a sample set.

3. **devkit's `guard.config.json` `structure` block** — two trees:
   - `cli` (`root: cli`, `entryAllowlist: index.mjs`, `libDomains.lib: [generate, husky, install]`,
     `commands` flat-kebab + `lib` domain-gated kebab modules).
   - `gate-engine` (`root: gate-engine`, `entryAllowlist: config.mjs`, `@root` domain-gate over the 8
     sub-engines incl. `structure`, `ignoredDirs: [eval]`).
   - `walls: []` (no import walls → the import-wall generator early-returns).

4. **Generated baselines** `eslint/baselines/{cli,gate-engine}.mjs` — both `[]` (devkit already
   conforms). Kept empty per the keep-empty-files rule (standing proof of zero exceptions).

5. **The eslint shim** `eslint.config.mjs` — a thin generated loader: reads `structure.trees`, loads
   each baseline, `createFolderStructure(compileToEslint(...))`. No topology encoded in it.

6. **Wiring.** `lint:structure` script (`eslint cli gate-engine`); a hard gate in `.husky/pre-commit`;
   and `.github/workflows/gate.yml` — the CI mirror (husky `--no-verify` is bypassable, research §2).

## Drift fixes found by dogfooding (each a walker↔eslint alignment)

- **`__tests__`** — the walker skips it (wall-free); the eslint rule must IGNORE it (`**/__tests__/**`),
  not enumerate allowed test shapes — else helpers/fixtures (`_helpers.mjs`) get rejected.
- **`ignoredDirs` depth** — the walker matches ignoredDirs by NAME at any depth; the emitted glob is
  `**/<dir>/**` (not `<dir>/**`) so nested `eval/` dirs are ignored identically.

## Deferred to frink-migration (ponytail-marked)

- **Preset trees** (`electron-renderer`/`electron-main` by name in `structure.trees[].preset`) — the
  frink walkers stay internal; exposing them as named presets is the last gap before frink can migrate.
- **Import-wall `wallClasses`** — `compileToEslint` covers folder-structure; the import-wall generator
  early-returns on empty `walls` but still uses the electron `DEFAULT_WALLS` shape for non-empty walls.
- **`devkit init` regeneration** of devkit's own baselines/shim — currently generated by invoking the
  library functions directly; folding into the `init` flow is mechanical follow-up.
