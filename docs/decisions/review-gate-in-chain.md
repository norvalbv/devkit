# review-gate-in-chain

## Target · 2026-07-02 — Reviewer agents run IN-CHAIN as headless claude -p judges spawned by the consumer's pre-commit hook

**Context:** Consumer repos enforced code review via root-agent-dispatched reviewer subagents whose approve.sh wrote .passed marker files checked by husky. The markers were agent-forgeable (touch .claude/.api-security-passed satisfies the gate with no review), stored diff hashes were never verified, and the root agent could skip dispatch entirely. The one part that WORKED was the checklist.mjs discipline: deterministic enumeration of review items (per staged file for commit-guard, per detected pattern category for domain reviewers) with a finalize that refuses pending items — benchmarked as the mechanism that stops a reviewer hallucinating that it validated units it never looked at.
**Ruling:** Reviewer agents run IN-CHAIN as headless claude -p judges spawned by the consumer's pre-commit hook via guard-review (gate-engine/review/). Domain selection is deterministic from guard.config.json topology; the judge is wrapped from the SAME synced .claude/agents/*.md brief used interactively; a block requires an opus-confirmed FAIL after a sonnet first pass; PASS verdicts cache by staged-diff SHA256 (.devkit/review-cache.json, git-common-dir anchored). The CHECKLIST DISCIPLINE IS RETAINED AND MECHANIZED: checklist.mjs scripts ship in devkit skills (report-only finalize), the gate wrapper mandates the generate/check-item/finalize workflow, the judge's allowlist grants exactly its own checklist script, and guard-review independently verifies the state-file artifact after the judge returns — a missing/incomplete/inconsistent checklist VOIDS a PASS to inconclusive (fail-open exit 2, never a cached pass). Only the forgeable half died: approve.sh, marker files, ship marker-carry. feature-completeness runs at commit-msg (message = intent), straight opus, WARN-by-default.
**Consequences:**
- Positive: Positive: review is un-forgeable (hook spawns judges; artifact is verified, not trusted) and per-item coverage is machine-checked, not claimed; identical diffs re-review free via the cache. Negative: commits in reviewer-scoped paths pay judge latency (parallel fan-out, 300s/420s ceilings under ship's 900s); a consumer without the claude binary loses review to fail-open where markers used to hard-require it.
- Negative: In-chain LLM judges CREATE a new hard-block class (previously only deterministic checks blocked). Accepted because the block is bounded: opus-confirmed only, artifact-verified, fail-open on outage, GUARD_NO_REVIEW bypass, deterministic trigger (domain paths staged).
**Vision-fit:** n/a — devkit internal guardrail architecture, not a product surface.

- 2026-07-06 — Gate hardening shipped from the frink PR #60 investigation (judge additions deferred pending gate-eval benchmarks — archive/correctness-judge-full holds the prototype): completeness gate fails CLOSED (exit 3) under GUARD_AI_STRICT on judge outage/timeout (stderr warnings are invisible to headless shipping agents; exit code is the only reliable channel), timeout 360s->420s (it SIGTERM'd and silently skipped on PR #60's branch-sized commit), full --stat map rides stdin ahead of the capped diff; review gate: a missing agent brief is INCONCLUSIVE with a sync nudge (never judged as an empty brief -> fake PASS), a FAIL echoes the judge's full transcript, GUARD_REVIEW_SKIP disables one reviewer by name.
- 2026-07-06 — Correctness reviewer added as the 6th gate agent (closes the bot-comment gap: 74% of real CodeRabbit/Macroscope findings are correctness/error-handling, previously uncovered). Built + measured on the new reviewer-eval bench (gate-engine/review/eval/reviewers/), whose corpus is REAL mined bot findings (anti-overfit, 41% held-out). Two measured decisions: (1) PER-REVIEWER model policy via Reviewer.model — correctness runs SINGLE-PASS haiku, NOT the cascade: the bench showed the opus escalation, which HELPS the domain reviewers (overturns their false-FAILs on decoys), SUBTRACTS correctness recall (0.78 first-pass -> 0.67 end-to-end — opus, framed to 'confirm or overturn', overturns subtle real bugs), and a cascade can't rescue a first-pass MISS anyway. (2) OVERRIDE VALVE — single-pass haiku is recall ~0.81 / precision ~0.80; a 0.80-precision HARD block false-blocks ~1-in-5 clean commits (devs disable it) and warn-only doesn't pre-empt the external bots (commit still lands -> CI -> bot comments). So a correctness FAIL BLOCKS, but each failed lens can be waived with a stated rationale (gate-engine/review/overrides.mts): fingerprint = sha12(reviewer + lens + sha256(reviewed diff)) — stable across re-runs, VOIDS when the code changes so a stale override can't suppress a new bug; waive via env OVERRIDE_<fp>_RATIONALE="…" (persisted through) or the committed .devkit/correctness-overrides.json (audit trail). All lenses waived -> PASS; any un-waived -> block, naming each fingerprint + the exact override line. Same shape as the dup-detection allowlist. Rationales are the signal to sharpen the reviewer (add a decoy) — precision improves from real use.
- 2026-07-07 — commit-guard (the pre-commit duplication agent) joins the per-reviewer model policy: pinned to SINGLE-PASS haiku (agents/commit-guard.md model:haiku, shipped #33/#34), NOT the opus-confirm cascade the blocking domain reviewers use. Rationale is the same shape as correctness-single-pass but for a DIFFERENT reason: commit-guard is ADVISORY over the deterministic matcher+clone gates (those are the blocking authority), so it never needs an opus confirm. Measured before pinning (bench in scratchpad/dup-bench, to persist): (1) FORENSICS — usage.db shows it had been running OPUS at ~$879/6wk (1.4% of ALL Claude spend), 63% pure-clean passes, and its unique catches over the free gates were thin. (2) RETRIEVAL is the real ceiling, not the model — a search-code vs codebase-memory bake-off over the co-occurrence allowlist labels (211 jscpd clones unbiased + 177 matcher pairs) scored ~0.59 recall@10 on verbatim clones and only ~0.25 on semantic dups; no engine swap helps, so the agentic judge sees the true partner <=25% of the time regardless of model. (3) JUDGE swap — a 28-pair labeled bench (haiku vs sonnet vs opus, identical prompt) had haiku agree with opus 96% and sonnet 100%; on the one genuinely-hard live look-alike (maskQuoted vs maskSingleQuoted, opposing security contracts) HAIKU was the only model that correctly said not-a-dup — opus and sonnet both false-alarmed. Opus is therefore STRICTLY DOMINATED (5x cost, zero measured quality gain, worse on the hard case). Roads not taken: DROP entirely — rejected because the free deterministic gates stay AND a ~$0 haiku surface still catches the occasional sub-threshold dup; KEEP OPUS — rejected as dominated; OPUS CASCADE like the blocking reviewers — rejected because commit-guard isn't a blocking gate. Anti-note: several of the old transcript-mined 'semantic dup' labels turned out ALREADY FIXED (e.g. bundled-claude-version.ts now imports parseVersionQuad; a shared resolveCliBinary now backs codex/cursor) — the agent's past findings were real and got consolidated. GAP: frink still pays opus because it is pinned to @norvalbv/devkit#v0.29.0 (pre-flip synced copy) — the saving only lands after a devkit release + frink pin bump.
- 2026-07-09 — conventions-reviewer added as the 7th gate agent (checks a consumer repo's own CLAUDE.md rules — never devkit's own — quoting rule+line or staying silent). Its AC forbids Bash (tools: Read,Grep,Glob only), so it cannot run the checklist.mjs workflow every other REVIEWERS entry depends on for anti-hallucination. Reviewer.skill/stateFile/cmds became OPTIONAL: a skill-less reviewer skips verifyChecklist entirely and trusts its verdict directly, the same trust level completeness.mts already uses for its own straight (non-cascaded) verdict — its substitute anti-hallucination mechanism is the AC's own quote-both-the-rule-and-the-line-or-stay-silent contract, enforced by the agent brief, not a checklist artifact. A false PASS here is fail-open/low-stakes (same as any other missed finding); a false FAIL is covered by the existing override valve (gate-engine/review/overrides.mts), extended to derive its lens from the parsed OFFENDING path:line (stable per violation) instead of checklist item names, since there is no checklist. Joins correctness-reviewer/commit-guard on the single-pass model-pin (model:'haiku'): no opus cascade, FAIL blocks directly. Domain 'conventions' reuses the 'all' root-union (rootsFor) but is exempt from the isSource/isTest filters selectReviewers applies to 'code'/'all' — a CLAUDE.md rule can govern any staged file type, not just source.
